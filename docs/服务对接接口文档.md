## 一、服务概览

CapCut Mate API 是一个基于 **FastAPI** 的剪映（CapCut）草稿自动化服务，功能覆盖草稿创建、各类素材编辑（视频/音频/图片/字幕/贴纸/特效/滤镜/遮罩/关键帧）、资源查询、云渲染导出。所有接口以 RESTful 风格提供，请求与响应均为 JSON。

**核心特性**：

- 通过 `draft_url` 串联整个生命周期：创建 → 编辑 → 保存 → 渲染。
- 同一草稿的写操作通过异步锁串行化，默认锁超时 30 秒。
- 草稿对象内存缓存，重启不丢失（持久化到磁盘）。
- 统一响应包装：业务异常以 HTTP 200 返回，`code != 0` 表示失败。

## 二、调用规约

### 2.1 基础信息

| 项目 | 值 |
|------|----|
| 协议 | HTTP / HTTPS |
| 默认端口 | `30000` |
| 路由前缀 | `/openapi/capcut-mate/v1` |
| Content-Type | `application/json; charset=utf-8` |
| 默认本地地址 | `http://localhost:30000` |
| 在线 Swagger | `http://<host>:30000/docs` |
| 在线 ReDoc | `http://<host>:30000/redoc` |

完整接口路径示例：`http://localhost:30000/openapi/capcut-mate/v1/create_draft`

### 2.2 统一响应格式

所有接口（包括异常）都以 HTTP 200 返回，业务结果通过 `code` 区分。中间件 `ResponseMiddleware` 自动包装：

**成功响应**

```json
{
  "code": 0,
  "message": "成功",
  "draft_url": "...",
  "track_id": "...",
  "其他业务字段": "..."
}
```

**失败响应**（不含业务字段）

```json
{
  "code": 2001,
  "message": "无效的草稿URL"
}
```

> 注意：失败响应没有 `data` 包裹层；成功响应也没有，业务字段直接平铺在顶层。这是本服务与传统 `{code, message, data}` 模式的关键差异。

### 2.4 时间与坐标单位

- **时间**：所有 `start` / `end` / `duration` 字段统一使用**微秒（μs）**，1 秒 = 1,000,000 μs。
- **坐标**：所有 `transform_x` / `transform_y` 偏移单位为**像素**，原点为画布中心，正值向右/向下。
- **缩放**：`scale_x` / `scale_y` 为乘数，`1.0` 即原始大小。
- **颜色**：均为十六进制字符串，如 `#ffffff`。

### 2.5 国际化（多语言响应）

通过 HTTP 请求头 `Accept-Language` 控制错误消息语言：

- `Accept-Language: zh` 或 `zh-CN`（默认）→ 返回中文消息
- `Accept-Language: en` 或 `en-US` → 返回英文消息

### 2.6 并发安全说明

- 所有写接口（add_*、save_draft、easy_create_material）按 `draft_id` 加异步锁。
- 锁超时：30 秒。同一草稿超过 30 秒未释放将抛出 `2042 DRAFT_LOCK_TIMEOUT`。
- 不同草稿之间互不阻塞，可水平并发。

## 四、草稿管理接口

### 4.1 创建草稿 create_draft

**功能**：创建一个新的剪映草稿项目并设置画布尺寸，是后续所有编辑操作的起点。

- **路径**：`POST /v1/create_draft`

#### 请求参数

| 参数 | 类型 | 必填 | 默认 | 范围/说明 |
|------|------|------|------|-----------|
| width | int | 否 | 1920 | ≥ 1，画布宽度（像素） |
| height | int | 否 | 1080 | ≥ 1，画布高度（像素） |

**常用画幅**：1920x1080（横屏 16:9）、1080x1920（竖屏 9:16）、1080x1080（方形 1:1）、3840x2160（4K）。

#### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| draft_url | string | 草稿 URL，后续所有操作必须传该字段 |
| tip_url | string | 项目帮助文档链接 |

#### 示例

```bash
curl -X POST "http://localhost:30000/openapi/capcut-mate/v1/create_draft" \
  -H "Content-Type: application/json" \
  -d '{"width":1080,"height":1920}'
```

```json
{
  "code": 0,
  "message": "成功",
  "draft_url": "http://localhost:30000/openapi/capcut-mate/v1/get_draft?draft_id=20231215123456abcd1234",
  "tip_url": "https://docs.jcaigc.cn/"
}
```

> draft_id 格式：14 位时间戳 + 8 位 UUID。`draft_url` 必须保存到对接服务侧，是草稿全生命周期的"句柄"。

---

### 4.2 保存草稿 save_draft

**功能**：将内存中的草稿修改持久化到磁盘。建议在每个 `add_*` 接口调用之后调用一次。

- **路径**：`POST /v1/save_draft`
- **并发**：异步锁，30 秒超时。

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| draft_url | string | 是 | `create_draft` 返回的 URL |

#### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| draft_url | string | 草稿 URL（与请求一致） |

---

### 4.3 获取草稿 get_draft

**功能**：返回草稿目录中所有文件的下载 URL，可用于将草稿导入剪映客户端。

- **路径**：`GET /v1/get_draft`

#### 请求参数（Query）

| 参数 | 类型 | 必填 | 长度 | 说明 |
|------|------|------|------|------|
| draft_id | string | 是 | 20–32 | 草稿 ID（可从 draft_url 中解析） |

#### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| files | string[] | 文件下载 URL 列表 |

#### 示例响应

```json
{
  "code": 0,
  "message": "成功",
  "files": [
    "http://host/drafts/<draft_id>/draft_info.json",
    "http://host/drafts/<draft_id>/draft_content.json",
    "http://host/drafts/<draft_id>/draft_meta_info.json"
  ]
}
```

## 五、素材添加接口

> 所有 `add_*` 接口都受异步锁保护，同一 `draft_id` 串行执行，30 秒超时。

### 5.1 添加视频 add_videos

**功能**：批量添加视频素材到草稿，支持遮罩、转场、变速、画中画等效果。

- **路径**：`POST /v1/add_videos`

#### 请求参数（顶层）

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| draft_url | string | 是 | - | 草稿 URL |
| video_infos | string | 是 | - | 视频信息数组的 **JSON 字符串**（注意是字符串，不是数组） |
| scene_timelines | array[object] | 否 | null | 场景时间线，与 `video_infos` 长度一致，用于变速 |
| alpha | float | 否 | 1.0 | 全局透明度 [0, 1] |
| scale_x | float | 否 | 1.0 | X 轴缩放，建议 [0.1, 5.0] |
| scale_y | float | 否 | 1.0 | Y 轴缩放，建议 [0.1, 5.0] |
| transform_x | int | 否 | 0 | X 轴偏移（像素） |
| transform_y | int | 否 | 0 | Y 轴偏移（像素） |

#### `video_infos` 元素结构（JSON 字符串解析后的每一项）

| 字段 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| video_url | string | 是 | - | 视频 URL，必须以 `http://` 或 `https://` 开头 |
| start | int | 是 | - | 时间轴起点（μs） |
| end | int | 是 | - | 时间轴终点（μs） |
| width | int | 否 | 自动 | 视频宽度，不传则自动获取 |
| height | int | 否 | 自动 | 视频高度，不传则自动获取 |
| duration | int | 否 | end - start | 视频文件总时长（μs） |
| mask | string | 否 | null | 遮罩类型：圆形/爱心/星形/矩形/线性/镜面 |
| transition | string | 否 | null | 转场名称 |
| transition_duration | int | 否 | 500000 | 转场时长，[100000, 2500000] |
| volume | float | 否 | 1.0 | 音量 [0, 10] |

#### `scene_timelines` 元素结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| start | int | 是 | 场景起点（μs） |
| end | int | 是 | 场景终点（μs） |

> **变速公式**：`speed = (video.end - video.start) / (scene.end - scene.start)`。视频时间线 0–2,000,000，场景时间线 0–1,000,000 → 2 倍速。

#### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| draft_url | string | 草稿 URL |
| track_id | string | 视频轨道 ID |
| video_ids | string[] | 视频素材 ID 列表 |
| segment_ids | string[] | 片段 ID 列表 |
| segment_infos | object[] | 片段详细信息：`{id, start, end}` |

### 5.2 添加音频 add_audios

**功能**：批量添加音频到独立的音频轨道。

- **路径**：`POST /v1/add_audios`

#### 请求参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| draft_url | string | 是 | 草稿 URL |
| audio_infos | string | 是 | 音频信息数组的 JSON 字符串 |

#### `audio_infos` 元素结构

| 字段 | 类型 | 必填 | 默认 | 说明 |

### 8.7 SRT 字幕 srt_infos

**功能**：下载并解析标准 SRT 字幕文件，自动提取文本与时间线（时间统一换算为微秒），生成可直接用于 `add_captions` 的字幕信息 JSON 字符串。相当于 `caption_infos` 的「SRT 入口」——省去手动拆分文本和时间线的步骤。

- **路径**：`POST /v1/srt_infos`

#### 请求参数

| 参数 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| srt_url | string(URL) | 是 | - | SRT 字幕文件 URL，必须以 `http://` 或 `https://` 开头 |
| font_size | int | 否 | null | 字体大小 |
| keyword_color | string | 否 | null | 关键词颜色（十六进制） |
| keyword_border_color | string | 否 | null | 关键词边框颜色 |
| keyword_font_size | int | 否 | null | 关键词字体大小 |
| keywords | string[] | 否 | null | 重点词列表，按字幕条目顺序一一对应 |
| in_animation | string | 否 | null | 入场动画名称 |
| in_animation_duration | int | 否 | null | 入场动画时长（μs） |
| loop_animation | string | 否 | null | 循环动画名称 |
| loop_animation_duration | int | 否 | null | 循环动画单次时长（μs） |
| out_animation | string | 否 | null | 出场动画名称 |
| out_animation_duration | int | 否 | null | 出场动画时长（μs） |
| transition | string | 否 | null | 转场名称 |
| transition_duration | int | 否 | null | 转场时长（μs） |

#### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| infos | string | 字幕信息 JSON 字符串，可直接作为 `add_captions` 的 `captions` 入参 |
| count | int | 解析出的字幕条目数量 |

#### 解析规则

- 兼容 UTF-8（含 BOM）/ GBK 编码、`\r\n` 与 `\n` 换行、毫秒分隔符为逗号或点。
- 单条字幕多行文本会合并为以 `\n` 连接的一段。
- 时间轴 `00:00:01,000 --> 00:00:04,074` 自动换算为微秒。
- 条目按开始时间升序排列；无时间轴或空文本的块自动跳过。
- 无法解析出任何有效条目时返回 `2018 无效的字幕信息`。

#### 示例

```bash
curl -X POST "http://localhost:30000/openapi/capcut-mate/v1/srt_infos" \
  -H "Content-Type: application/json" \
  -d '{
    "srt_url": "https://example.com/subtitles/demo.srt",
    "font_size": 15,
    "keywords": ["重点词", "关键句"],
    "keyword_color": "#ff7100",
    "in_animation": "淡入"
  }'
```

```json
{
  "code": 0,
  "message": "成功",
  "infos": "[{\"start\":0,\"end\":2000000,\"text\":\"你好，世界\",\"font_size\":15}]",
  "count": 1
}
```

#### 典型用法

```
1. srt_infos          # 传 SRT URL，得到 infos JSON 字符串
2. add_captions       # 将 infos 作为 captions 入参落入草稿
3. save_draft
```

## 九、典型工作流

### 9.1 基础视频创建

```
1. create_draft         # 创建草稿，记录 draft_url
2. add_videos           # 添加视频
3. add_audios           # 添加 BGM
4. add_captions         # 添加字幕
5. save_draft           # 持久化
6. gen_video            # 提交渲染
7. gen_video_status     # 轮询直到 completed
```

### 9.2 完整智能编辑流程

```
1. create_draft         # 设置画布
2. audio_timelines      # 根据音频生成时间线
3. video_infos / imgs_infos / caption_infos  # 拼装 *_infos JSON
4. add_videos / add_images / add_captions    # 落入草稿
5. add_filters / add_effects                 # 视觉效果
6. add_keyframes        # 关键帧动画
7. add_audios           # 配音
8. add_sticker          # 装饰
9. save_draft
10. gen_video
11. gen_video_status    # 轮询
```

### 9.3 调用注意事项

1. **JSON 字符串字段**：`video_infos`/`audio_infos`/`image_infos`/`captions`/`effect_infos`/`filter_infos`/`keyframes` 这几个字段都是 **嵌套 JSON 字符串**，不是数组——传入时需 `JSON.stringify(...)` 后作为字符串放入请求体。
2. **URL 校验**：`*_url` 字段必须以 `http://` 或 `https://` 开头，否则在 schema 层就会被拒绝（错误码 1001）。
3. **每次写操作后 save**：编辑接口只在内存修改，必须显式调用 `save_draft` 才会落盘，否则 `gen_video` 拿不到最新内容。
4. **锁超时**：同一草稿写操作串行，超过 30s 等待会返回 `2042 DRAFT_LOCK_TIMEOUT`，调用方应做退避重试。
5. **画布坐标**：`transform_x/y` 以**画布中心为原点**，正值向右/向下；底层会根据当前画布宽/高做归一化。
6. **`apiKey`**：仅在云渲染（`gen_video`）需要；启用与否由服务端 `ENABLE_APIKEY` 决定。
7. **`gen_video` 平台限制**：仅 Windows 部署可用（依赖剪映 PC 客户端）。
8. **响应包装**：所有响应一律 HTTP 200，`code != 0` 即业务失败；不要根据 HTTP 状态码判断业务结果。
